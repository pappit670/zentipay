# Zenti — Storytelling, Chat, NFC & Positioning (2026-06-15)

> Companion to EXPANSION-VISION.md. The big frame: **Zenti = a Financial Operating System**.
> Zenti Pay *creates* activity; Zenti Business *organizes* it. Payments → Records → Reports →
> Payroll → Compliance → Growth, all connected.

## Positioning
- Internal: **"Zenti Business is the purifier."** Businesses already have M-Pesa, tills, paybills, POS,
  invoices, suppliers, payroll. The problem isn't *accepting* payments — it's chaos, missing records,
  manual tracking, manual reports. Zenti sits **above** the funnel and turns **transactions → clarity**.
- External one-liner: **"Keep the tools you already use. Make payments easier."**
- Strongest framing: not a wallet, not a POS — a **Financial Operating System**.

## Website direction — STORY over features
Stop the feature-grid. For each capability, tell a 4-beat story:
1. **The problem** — "Business owner manually checks payments."
2. **The frustration** — "Missed transactions. Late nights. Spreadsheets."
3. **The solution** — "Zenti organizes everything automatically."
4. **The outcome** — "More time. Less stress. Real visibility."
(We already started this: pain → before/after → benefits. Push it further per-feature.)

### Plain, human language (for mainstream users)
Avoid: "Revenue optimization", "Transaction intelligence", "Financial infrastructure".
Use: **Money Received · Money Spent · Money Left · Money for KRA · Money for Suppliers.**

### Video strategy (HyperFrames / Higgsfield)
Every major feature gets a short cinematic clip of the real flow instead of paragraphs. e.g. Payment link:
*Friend requests → notification → tap → pay → success.* (First HyperFrames reel shipped: `web/hyperframes/zenti-reel`.)

## NFC payment flow (the winning flow)
- **Register once:** merchant gets NFC tag → Zenti Business → Settings → Register NFC Tag → tag is
  permanently linked to the merchant account.
- **Customer:** input amount → tap merchant tag → merchant identified → Face ID / fingerprint → success.
  No till entry, no merchant search, no wrong-recipient errors. **Input. Tap. Pay.**
- **Small biz:** input → tap → pay. **Large biz:** POS generates amount → Zenti API → customer taps → pay.
- **POS API:** don't compete with Cashless/restaurant/pharmacy/retail POS — give them a **Zenti Payments
  API** to generate payment sessions. Zenti = infrastructure.

## In-app chat — "money conversations" (NOT a messenger)
- **Money first, messages second.** Every thread revolves around requests, payments, tips, split bills, repayments.
- **P2P:** search phone / @ztag / username → create request or payment → recipient gets notification →
  open chat → accept → pay. All inside Zenti (no WhatsApp number + screenshot dance).
- **Killer use case — discreet tips:** restaurant bars direct tips; waiter shows "My QR"; customer scans →
  Zenti chat → tip → done. No awkwardness, no cash. Creates a **micro-economy** (tips, gifts, requests,
  repayments) inside Zenti.
